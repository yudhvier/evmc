5 ## An example
6 
7 You can start with [the example implementation of EVMC VM interface in C](@ref example_vm.c).

9 ## VM instance
10 
11 The VM instance is described by the ::evmc_vm struct. It contains the
12 basic static information about the VM like name and version. The struct also
13 includes the VM methods (in form of function pointers) to allow the Host
14 to interact with the VM.
15 
16 Some methods are optional. The VM must implement at least all mandatory ones.
17 
18 The instance struct must also include the EVMC ABI version (::EVMC_ABI_VERSION)
19 it was build with. This allows the Host to check the ABI compatibility when
20 loading VMs dynamically.
21 
22 The VM instance is created and returned as a pointer from a special "create"
23 function. The EVMC recommends to name the function by the VM codename,
24 e.g. ::evmc_create_example_vm().
25 
26 ## VM methods implementation
27 
28 Each VM methods takes the pointer to the ::evmc_vm as the first argument.
29 The VM implementation can extend the ::evmc_vm struct for storing internal
30 data. This allow implementing the VM in object-oriented manner.
31 
32 The most important method is ::evmc_vm::execute() because it executes EVM code.
33 Remember that the Host is allowed to invoke the execute method concurrently
34 so do not store data related to a particular execution context in the VM instance.
35 
36 Before a client can actually execute a VM, it is important to implement the three
37 basic fields for querying name (::evmc_vm::name), version (::evmc_vm::version)
38 and capabilities (::evmc_vm::get_capabilities()) as well as the ::evmc_vm::destroy()
39 method to wind the VM down.
40 
41 Other methods are optional.
42 
43 ## Resource management
44 
45 All additional resources allocated when the VM instance is created must be
46 freed when the destroy method is invoked.
47 
48 The VM implementation can also attach additional resources to the ::evmc_result
49 of an execution. These resource must be freed when the ::evmc_result::release()
50 method is invoked.
